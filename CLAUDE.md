# My Weight - Workout & Weight Loss Tracker

## Опис проекту
Простий tracker для відстеження тренувань, ваги та прогресу в схудненні. Використовує vanilla JavaScript, без фреймворків.

## Структура проекту
- `index.html` - головний файл з workout планом та UI
- `PROFILE.md` - профіль користувача (зріст, початкова вага, ціль)
- `PROGRESS.md` - прогрес ваги
- `EXERCISES.md` - опис вправ
- `NUTRITION.md` - план харчування
- `FOODS.md` - список продуктів
- `WEEKLY-MENU.md` - меню на тиждень
- `NOTES.md` - загальні нотатки
- `logs/` - директорія для логів тренувань

## Workout план
- **Понеділок**: Відпочинок
- **Вівторок**: Кардіо (30-40 хв)
- **Середа**: Силові - верх тіла
- **Четвер**: Кардіо (30-40 хв)
- **П'ятниця**: Силові - низ + корпус
- **Субота**: Кардіо (30-40 хв)
- **Неділя**: Силові - все тіло

## Щоденний чек-ін

### ВАЖЛИВО: Інтерактивний режим
**Користувач хоче вручну вводити дані через діалог з Claude.**
Claude має ЗАВЖДИ питати кожен параметр окремо, чекати відповіді, і потім записувати.

### Workflow:
**Блок 1 — Вага та метрики тіла:**
1. Питати: **"Яка вага була вранці?"** → чекати відповідь
2. Питати: **"Body Fat %?"** → чекати відповідь
3. Питати: **"Віцеральний жир?"** → чекати відповідь
4. Питати: **"Маса жиру в організмі?"** (кг) → чекати відповідь
5. Питати: **"Muscle %?"** → чекати відповідь

**Блок 2 — Сон (дані з Apple Watch):**
6. Питати: **"Дані сну з Apple Watch?"** → чекати відповідь
   - Тривалість сну (год:хв)
   - Оцінка: Duration (X/50), Bedtime (X/30), Interruption (X/20)
   - Фази: REM, Core, Deep
   - Пульс уві сні: min/max (bpm)
   - Частота дихання: min/max

**Блок 3 — Тренування:**
7. Питати: **"Чи було тренування сьогодні?"** → чекати відповідь
   - Якщо **ТАК** → збирати дані про тренування (питання по черзі)
   - Якщо **НІ** → перейти до їжі

**Блок 4 — Їжа (можна відкласти на пізніше):**
8. Питати: **"Що їв сьогодні?"** → чекати відповідь
   - Сніданок: що їв? Вага страв (г) якщо зважив
   - Обід: що їв? Вага страв (г) якщо зважив
   - Вечеря: що їв? Вага страв (г) якщо зважив
   - Перекуси (якщо були): що? Вага (г)
   - **ВАЖЛИВО:** Протягом першого тижня (17-23.02) трекаємо вагу страв для точного розрахунку калорій
   - Розрахувати точні калорії на основі ваги
   - Якщо користувач каже "пізніше" → записати що є і повернутись до їжі коли скаже

### Для тренування питати по черзі:
1. **Який тип тренування?** (Силові / Кардіо)
2. **Скільки тривало?** (хв)
3. **Як почувався? (1-5)**
4. **Є дані з Apple Watch?**
   - Якщо так → попросити надати (калорії, пульс, тощо)
5. **Чи є коментарі/нотатки?**
6. Для силових: **Які вправи виконав?** (можна список)
7. Для кардіо: **Який тип кардіо?** (Велотренажер / Shadow Boxing / Скакалка)

## Трекінг тренувань

### Дані для збереження
Коли користувач каже "було тренування", треба зібрати:

1. **Базова інформація**:
   - Дата (YYYY-MM-DD)
   - День тижня
   - Тип тренування (Силові / Кардіо / Відпочинок)
   - Вага вранці (кг)

2. **Для силових тренувань**:
   - Список виконаних вправ
   - Для кожної вправи:
     - Назва вправи
     - Вага гантелей (кг)
     - Кількість підходів × повторів (наприклад: "3 × 12")
     - Чи виконано повністю
   - Загальна тривалість (хв)
   - Самопочуття (1-5)
   - Коментарі/нотатки (опціонально)

3. **Для кардіо тренувань**:
   - Тип кардіо (Велотренажер / Shadow Boxing / Скакалка)
   - Тривалість (хв)
   - Середній пульс
   - Максимальний пульс
   - Самопочуття (1-5)
   - Коментарі/нотатки (опціонально)

### Дані з Apple Watch
Користувач може надати дані з Apple Watch. Зберігати:
- **Тривалість тренування** (хв)
- **Активні калорії** (kcal)
- **Загальні калорії** (kcal)
- **Середній пульс** (bpm)
- **Максимальний пульс** (bpm)
- **Середній темп** (для кардіо, якщо є)
- **Дистанція** (км, якщо є)
- **Зони пульсу** (час у кожній зоні, якщо є)

### Формат зберігання
Лог-файли у форматі JSON або markdown у директорії `logs/`:
- `logs/2026-02.json` - всі тренування за лютий 2026
- або `logs/workout-2026-02-17.json` - окреме тренування

## UI Фічі для додавання
1. Кнопка "Записати тренування" на сторінці кожного дня
2. Форма для введення даних про тренування
